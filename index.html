üì± What is Flutter?

Flutter is Google's open-source UI toolkit for building beautiful, natively compiled applications for mobile, web, and desktop from a single codebase.

üß© Key Concept

Everything in Flutter is a Widget! Widgets are the building blocks of your UI - from buttons and text to layouts and animations.

Why Choose Flutter?

  • Fast Development - Hot reload sees changes instantly
  • Beautiful UI - Rich set of customizable widgets
  • Native Performance - Compiles to native ARM code
  • Single Codebase - iOS, Android, Web, Desktop from one code

üß© Widget Fundamentals

üì¶ Understanding Widgets

Widgets describe what their view should look like. Flutter builds the UI by composing simple widgets into complex ones.

StatelessWidget - Immutable UI

Use when UI doesn't change after creation (static content)

import 'package:flutter/material.dart'; class Greeting extends StatelessWidget { final String name; Greeting({required this.name}); @override Widget build(BuildContext context) { return Text('Hello, $name!'); } }

StatefulWidget - Dynamic UI

Use when UI needs to change based on user interaction or data updates

class Counter extends StatefulWidget { @override _CounterState createState() => _CounterState(); } class _CounterState extends State<Counter> { int count = 0; void increment() { setState(() { count++; // Update state }); // Triggers rebuild } @override Widget build(BuildContext context) { return Column( children: [ Text('Count: $count'), ElevatedButton( onPressed: increment, child: Text('Increment'), ), ], ); } }
‚ö†Ô∏è setState() Explained

Always wrap state changes in setState(). This tells Flutter to rebuild the widget with new data. Without it, UI won't update!

üì± Building a Complete App

Basic App Template

import 'package:flutter/material.dart'; void main() { runApp(MyApp()); } class MyApp extends StatelessWidget { @override Widget build(BuildContext context) { return MaterialApp( title: 'My App', theme: ThemeData( primarySwatch: Colors.blue, brightness: Brightness.light, ), home: HomePage(), debugShowCheckedModeBanner: false, ); } } class HomePage extends StatelessWidget { @override Widget build(BuildContext context) { return Scaffold( appBar: AppBar( title: Text('Home'), actions: [ IconButton( icon: Icon(Icons.settings), onPressed: () {}, ), ], ), body: Center( child: Text('Hello Flutter!'), ), floatingActionButton: FloatingActionButton( onPressed: () {}, child: Icon(Icons.add), ), bottomNavigationBar: BottomNavigationBar( items: [ BottomNavigationBarItem( icon: Icon(Icons.home), label: 'Home', ), BottomNavigationBarItem( icon: Icon(Icons.search), label: 'Search', ), BottomNavigationBarItem( icon: Icon(Icons.person), label: 'Profile', ), ], ), ); } }

üîÑ State Management

setState() - For Simple State

‚è±Ô∏è Asynchronous Programming

Future & Async/Await

// Simple async function Future<String> fetchData() async { await Future.delayed(Duration(seconds: 2)); return 'Data loaded!'; } // Using async/await void loadData() async { print('Loading...'); String data = await fetchData(); print(data); } // Error handling void loadDataWithError() async { try { String data = await fetchData(); print(data); } catch (e) { print('Error: $e'); } }

üêõ Debugging & Best Practices

Hot Reload Commands

While app is running in terminal:

  • r - Hot reload (apply changes)
  • R - Hot restart (full restart)
  • h - Help (show all commands)
  • q - Quit
  • d - Detach (keep app running)
  • P - Toggle performance overlay

Common Issues & Solutions

üîß Build Errors?

Try flutter clean then flutter pub get. This fixes 90% of weird build issues!

üì± Widget Overflow?

Use SingleChildScrollView or ListView to make content scrollable.

üîÑ UI Not Updating?

Make sure you're calling setState() when changing state in StatefulWidget!

Best Practices

  • ‚úì Use const constructors when possible for better performance
  • ‚úì Extract widgets to keep build methods small and readable
  • ‚úì Use meaningful variable and widget names
  • ‚úì Handle null safety properly with ? and !
  • ‚úì Use async/await for asynchronous operations
  • ‚úì Test on multiple devices and screen sizes
  • ‚úì Follow Flutter's Material Design or Cupertino guidelines
